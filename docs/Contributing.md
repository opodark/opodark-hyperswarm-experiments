# Contributing

## Regole semplici

- Una feature per branch.
- Un refactor per branch.
- Niente file dump dentro la repo.
- Niente rename non coordinati.

## Checklist PR

- Build/test locali ok.
- Nessun import rotto.
- Nessun path vecchio lasciato in giro.
- README e wiki aggiornati.

## Flusso consigliato

1. Lavora nella repo experimental.
2. Fai commit piccoli.
3. Push frequente.
4. Solo dopo porta le modifiche nel main repo.
